#+TITLE: Typtel - Typing Telemetry
#+AUTHOR: Aayush Bajaj

* Overview

=typtel= is a keystroke and mouse distance metrics tool for developers. It tracks every keypress including modifier keys, escape sequences, and shortcuts - giving you accurate metrics of your actual typing and mousing activity.

* Features

- *Keystroke Counting*: Tracks every keypress, not just completed words
- *Word Counting*: Detects word boundaries for accurate word counts
- *Mouse Distance*: Records mouse distance (in feet/miles) and click counts
- *Menu Bar Integration*: Shows daily stats in macOS menu bar
- *Settings Menu*: Configure what stats to display in the menu bar
- *Stillness Leaderboard*: Track your least-mousey days
- *Typing Speed Test*: Built-in WPM test with Monkeytype-style options
- *Charts & Heatmaps*: Visualize your typing patterns over time
- *SQLite Storage*: Persistent local storage of all metrics

* Installation

#+begin_src shell
brew tap abaj8494/typing-telemetry
brew install typing-telemetry
#+end_src

* Setup

** 1. Grant Accessibility Permissions

=typtel= requires accessibility permissions to capture keystrokes and mouse movements:

1. Open *System Settings* > *Privacy & Security* > *Accessibility*
2. Click the *+* button
3. Press =Cmd+Shift+G= and paste: =~/Applications/Typtel.app=
4. Enable the checkbox for Typtel
5. Restart the service (see below)

** 2. Start the Service

#+begin_src shell
brew services start typing-telemetry
#+end_src

The menu bar app will now run automatically on login.

* Usage

** Menu Bar

The menu bar shows your daily stats. Click it to see:
- Today's and this week's keystrokes, words, clicks, and mouse distance
- Charts (weekly/monthly overviews with heatmaps)
- Stillness Leaderboard (days with least mouse movement)
- Settings (configure what to display)

** CLI Commands

#+begin_src shell
typtel              # Interactive TUI dashboard
typtel stats        # Show statistics
typtel today        # Today's keystroke count
typtel test         # Typing speed test
typtel test -w 50   # Test with 50 words
#+end_src

** Typing Test

| Key       | Action                    |
|-----------+---------------------------|
| =tab=     | Restart with new words    |
| =esc=     | Open options menu         |
| =enter=   | Start new test            |
| =ctrl+c=  | Quit                      |

Options (press =esc=): Layout emulation, live WPM, test length, uppercase, punctuation, pace caret.

* reMarkable Connection

=typtel= can optionally accept keystroke aggregates from an *external device* -
for example a reMarkable Paper Pro with a Type Folio - and store them as a
/separate/ feed alongside your Mac's stats. *This is entirely opt-in and
disabled by default.* A fresh install (or a =brew upgrade=) never opens a port
or talks to any device until you explicitly run =typtel devices enable=. If you
don't have a device to feed it, you can ignore this section completely.

The transport is a token-gated HTTP listener bound to *loopback only*
(=127.0.0.1:8889=), reached from the device over your private *Tailscale*
tailnet. Device stats never mix into your Mac totals - they're queryable on
their own (=typtel today --device <id>=) and surface as an optional ="devices"=
block in =typtel stats --json=.

** On the Mac (this app)

1. Enable the ingest API and grab the bearer token:

   #+begin_src shell
   typtel devices enable        # generates a token, prints it + the bind addr
   #+end_src

   The token is also printable later with =typtel devices token= (=--rotate= to
   regenerate). Toggling enable/disable requires a *menubar restart* to take
   effect.

2. Publish the loopback listener to your tailnet with =tailscale serve= (raw TCP
   passthrough - keeps the device on plain HTTP, no TLS/MagicDNS needed):

   #+begin_src shell
   tailscale serve --bg --tcp 8889 tcp://127.0.0.1:8889
   tailscale serve status       # verify; reset with: tailscale serve --tcp=8889 off
   #+end_src

   Binding loopback + =tailscale serve= is deliberate: on macOS the Tailscale
   client won't deliver inbound tailnet connections to a listener on the utun IP,
   and this keeps the port off your LAN. The bearer token is the auth boundary.

** On the device

The device PUTs *absolute* daily totals (not deltas, so a retried PUT over a
flaky link can never double-count) to:

#+begin_src text
PUT http://<your-mac-tailnet-ip>:8889/v1/devices/<id>/days/<YYYY-MM-DD>
Authorization: Bearer <token>
{ "keystrokes": …, "letters": …, "modifiers": …, "special": …,
  "words": …, "active_ms": … }
#+end_src

Probe liveness (no auth) with =GET /v1/health=. The device classifies its own
keys and sends pre-aggregated counts - =typtel= stores them as opaque totals and
never re-classifies them.

#+begin_quote
*reMarkable gotcha:* the tablet's =tailscaled= runs in userspace-networking mode
(no =/dev/net/tun=), so its own processes can't open a socket directly to a
=100.x= peer. The device must send its PUTs through tailscaled's netstack (an
=--outbound-http-proxy-listen= proxy, or =tailscale nc=), not a plain
=curl=/=requests.put=. That's configured on the device side.
#+end_quote

** Inspecting and removing device data

#+begin_src shell
typtel devices               # list registered devices + last-seen
typtel devices show <id>     # recent days for a device (add --json)
typtel devices forget <id>   # delete a device and all its recorded days
typtel devices disable       # turn the listener off (restart menubar to apply)
#+end_src

* Data Storage

All data is stored locally in =~/.local/share/typtel/=:
- =typtel.db=: SQLite database
- =logs/=: Application logs

* Privacy

All data stays on your machine. Nothing is sent externally.

* Uninstalling

#+begin_src shell
brew services stop typing-telemetry
brew uninstall typing-telemetry
rm -rf ~/.local/share/typtel  # Optional: remove data
#+end_src
